Turn earnings calls, strategy decks, and internal memos into queryable causal graphs

Financial analysts dedicate hours reading reports to understand the causal relationships driving revenue, risk, and growth. MarketFoundry treats unstructured text as a first-class data source, automatically building custom knowledge graphs that answer "Why did Q3 miss?" or "What caused churn?" with evidence from across your entire document corpus.

The Problem We Solve

The Challenge

Financial decision-making relies heavily on unstructured text data‚Äî earnings call transcripts, strategy memos, press releases‚Äîyet this qualitative information is just as critical as numerical data. Causal claims like "higher pricing drove 15% revenue lift" or "omnichannel expansion accelerated branded TPV growth" contain the why behind the numbers.

Analysts dedicate countless hours manually reading through hundreds of pages across PDFs, Word documents, and HTML to piece together causal relationships. This manual process doesn't scale, and valuable insights remain scattered across document silos.

Our Solution

We built a document-aware pipeline with custom extraction configurations for every financial document type. Each document‚Äîwhether an earnings call, strategy deck, or internal memo‚Äîgets intelligently classified, sectioned, and routed to specialized schemas that capture the most accurate causal relationships. The result: a unified knowledge graph treating your unstructured text as rigorously as numerical data.

How It Works

MarketFoundry uses a document-aware pipeline that intelligently processes each financial document type to extract the most valuable causal relationships.

üìä Architecture Diagram

Pipeline: Upload ‚Üí Classify ‚Üí Section ‚Üí Extract ‚Üí Graph

1

Document Ingestion

Upload PDF, DOCX, HTML, or TXT files. Our system handles any financial document format.

python src/run.py --file your_docs/
2

Document Classification

ML model automatically detects document types: Earnings Calls, Strategy Decks, Internal Memos, SEC Filings, Technical Reports, etc.

3

Intelligent Sectioning

Each document type is sectioned differently. We're relying on LLMs to capture as valuable information as possible from each document.

4

Schema Routing

5

Causal Extraction

OneKE + Qwen2.5-7B extracts structured causal triples (subject, predicate, object) with confidence scores and source citations.

6

Knowledge Graph

All triples are stored in Neo4j, creating a unified causal graph you can query to gain a better understanding of the causal relationships.

Built With
